## 📌 About The Repository

This repository contains practical implementations, concepts, and examples of the **Spring AI Framework** using:

- OpenAI Models
- Ollama Local Models
- Prompt Engineering
- Prompt Templates
- Advisors & Custom Advisors
- Multi Model Integration
- Chat Client API
- AI Service Architecture

The main goal of this repository is to provide structured and practical learning for developers who want to build AI-powered Java applications using Spring Boot and Spring AI.

---

# 📂 Repository Structure

## 📁 Projects Included

### 📌 Chat-Client-API-Project
Implementation of Spring AI Chat Client API with request/response handling and prompt execution.

### 📌 Multi-Model-Project
Demonstrates how to integrate and switch between multiple AI models such as:
- OpenAI
- Ollama

### 📌 Ollama-Ai-First-Project
Basic Spring AI integration using locally running Ollama models.

### 📌 OpenAiFirstProject
Getting started with OpenAI integration in Spring AI.

### 📌 Spring-Ai-Advisors
Implementation of built-in Spring AI Advisors.

### 📌 Spring-AI-Custom-Advisors
Custom advisor implementation for modifying prompts and responses dynamically.

### 📌 Spring-Ai-Prompt-Templating
Introduction to Prompt Template concepts using Spring AI.

### 📌 Spring-Ai-Prompt-Templating-Part2
Advanced prompt templating concepts with dynamic variables and reusable templates.

# ⚙️ Prerequisites

Before running the projects, make sure you have:

- Java 17 or above
- Maven installed
- STS / IntelliJ IDEA
- OpenAI API Key
- Ollama Installed (for local LLM projects)

---

# 🔑 OpenAI Configuration

Add your OpenAI API key inside:

```properties
spring.ai.openai.api-key=YOUR_API_KEY
```

---

# 🖥️ Ollama Setup

Install Ollama from:

```text
https://ollama.com/
```

Run a model locally:

```bash
ollama run llama3
```

---

# ▶️ Running The Project

## Clone Repository

```bash
git clone https://github.com/abhishekkargeti1/Spring-Ai-Notes.git
```

## Navigate to Project

```bash
cd Spring-Ai-Notes
```

## Run Using Maven

```bash
mvn spring-boot:run
```
